## 🎓 ARCHITECTURE NOTES

### Why GetClient() is Public
- MonitoringService needs direct access for channel name modifications
- Discord.Net doesn't provide abstraction for channel naming
- Marked with WARNING to discourage general use
- All other operations use public methods (safer)

### Why DiscordService is a Wrapper
- Provides simplified interface for internal services
- Abstracts away Discord.Net details
- Allows for easy switching of Discord library if needed
- Current implementation delegates to DiscordBotService

### Thread Safety
- All Discord operations are async
- Plugin uses Torch.MySandboxGame.Static.Invoke for game world access
- Timers run on separate threads, properly synchronized
- Event handlers are async-aware
